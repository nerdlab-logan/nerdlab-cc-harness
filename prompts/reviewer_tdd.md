# TDD 모드 — 리뷰어 보조 룰

## 테스트 누락 = blocking 격상

TDD 모드에서 다음 상황은 **severity: minor 가 아닌 severity: blocking** 으로 분류한다:

- 새로 추가된 기능·함수·클래스에 대응하는 테스트가 없는 경우
- 테스트가 있으나 실제로 실패→통과 사이클을 거쳤다는 증거가 coder 출력에 없는 경우
- 기존 테스트를 삭제하거나 skip 처리해서 통과시킨 경우

## 검증 항목

리뷰 시 아래 항목을 반드시 확인한다:

1. coder 출력에 "테스트 먼저 작성" 선언이 있는가?
2. 추가된 테스트 케이스 이름이 구현 파일의 함수·클래스와 대응하는가?
3. plan 의 검증 명령 실행 결과(또는 그에 준하는 증거)가 포함되어 있는가?

## 판정 규약

테스트 누락이 확인되면 `status: needs-fix` 와 함께 해당 issue 의 severity 를 `blocking` 으로 설정한다.
테스트는 있으나 통과 증거가 불명확하면 severity `blocking` 으로 분류하고 coder 에게 재확인을 요청한다.
