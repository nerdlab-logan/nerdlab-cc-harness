# TDD 모드 — 코더 보조 룰

## 필수 사이클

구현 순서를 반드시 아래와 같이 따른다:

1. **실패 테스트 먼저 작성** — 아직 존재하지 않는 동작을 검증하는 테스트를 먼저 작성한다. 이 시점에서 테스트는 실패해야 한다.
2. **최소 구현** — 테스트를 통과시키는 데 필요한 최소한의 코드만 작성한다.
3. **통과 확인** — plan 의 검증 명령을 실행해 작성한 테스트가 실제로 통과하는지 확인한다.

## 금지 사항

- 테스트 없이 구현 코드를 먼저 제출하지 않는다.
- 통과를 확인하지 않은 테스트를 "작성했다"고 보고하지 않는다.
- 기존 테스트를 삭제하거나 수정해서 통과시키지 않는다.

## 출력 규약

coder stdout 첫 번째 섹션에 "테스트 먼저 작성 완료 후 구현" 흐름을 따랐음을 명시한다.
테스트 파일 경로와 추가한 테스트 케이스 이름을 한 줄씩 열거한다.
